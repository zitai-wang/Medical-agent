# 边界情况、良恶性鉴别与冲突校验（edge_cases.md）

本文件定义提取器在**非规范文本、信息冲突、良恶性未定**等复杂场景下的处理规则，是赛道2"鲁棒性"与"一致性"考核的核心。

---

## 一、良恶性鉴别规则（综合判定，不擅断）

判定 `benign_malignant` 时，按**金标准优先级**综合，并在 `bm_basis` 列出依据：

### 1.1 金标准优先级
```
病理诊断（确诊） > 临床确诊 > 影像典型表现 > 影像疑似/未定性
```
- **有病理** → 以病理为准。即使影像描述模糊（如患者D"考虑FNH，恶性不除外"），病理确诊FNH则判**良性**，编 D13.4。
- **无病理，仅影像** → 按影像定性程度记录为"疑似恶性/疑似良性/性质待定"，**绝不输出确定性恶性/良性**。

### 1.2 支持恶性（HCC）的征象
| 维度 | 支持HCC |
|------|---------|
| 影像 | 动脉期强化+门脉/延迟期廓清（快进快出）、假包膜、门脉癌栓、肝硬化背景 |
| 病理 | 肝细胞癌、肝板增厚>2-3层、网状纤维断裂 |
| 免疫组化 | Glypican-3(+)、HSP70(+)、GS弥漫(+)（三联≥2项阳性） |
| 标志物 | AFP↑、AFP-L3↑、PIVKA-II↑ |

### 1.3 支持良性（FNH）的征象
| 维度 | 支持FNH |
|------|---------|
| 影像 | 地图状/轮辐状强化、中央瘢痕、肝胆期等/高信号、无假包膜 |
| 病理 | 正常肝小叶结构消失但肝细胞形态正常、肝板≤2层、纤维间隔含厚壁血管+小胆管增生 |
| 免疫组化 | Glypican-3(-)、GS**地图状**(+)、CD34部分区域毛细血管化、网状纤维**无断裂** |
| 标志物 | AFP正常 |

> **关键**：GS"弥漫阳性"支持HCC，"地图状阳性"支持FNH——同一标记不同模式意义相反，提取时**必须保留"地图状/弥漫"修饰词**，不可简化为"GS(+)"。

### 1.4 未定性表述的标准化记录
| 原文表述 | 记录为 |
|---------|--------|
| "考虑××" | 疑似××（置信度中） |
| "××不能完全除外""恶性不除外" | ××待排（置信度低） |
| "××可能" | 疑似××（置信度中） |
| "建议结合临床/进一步检查" | 标注"需补充检查方可定性" |
| 病理"支持为××" | ××（病理倾向，高置信度） |

## 二、信息冲突处理

| 冲突类型 | 示例 | 处理方式 |
|---------|------|---------|
| 不同文书诊断不一致 | 门诊"血管瘤?" vs 病理"HCC" | 按可信度优先级（病理优先）取值；在 warnings 记录"诊断演变" |
| 影像 vs 病理矛盾 | 影像"考虑HCC" vs 病理"FNH" | 以病理为准，warnings 标注影像-病理差异 |
| 分期与征象矛盾 | 描述"早期"但有门脉癌栓 | 按征象判 CNLC Ⅲa，warnings 标注矛盾 |
| 免疫组化内部矛盾 | 标HCC但 Hep(-)/GPC-3(-) 全阴 | 不擅改，warnings 标注"免疫表型不典型，建议复核" |
| 同一指标多次结果不同 | AFP 多次检验值不同 | 取最近一次或全部列出，标注时间（如有） |
| TNM/M1 与总分期不符 | M1 但写 II 期 | 按 M1 校正为 IV 期，warnings 记录 |

> 原则：**发现冲突一律记录到 warnings，绝不擅自删除任一来源信息**。

## 三、缺失与脱敏处理

| 情形 | 处理 |
|------|------|
| 字段标"已脱敏" | 该字段 `value="未提及"`，不视为有效值 |
| "图片中该项未见明确填写内容" | `value="未提及"` |
| "人未至"（体格检查） | 体征 `value="未提及（人未至）"` |
| 只有门诊病历无病理/影像 | 病理/影像类字段整体标"本次输入未包含此类文书"，**不计入覆盖度分母** |
| 关键诊断字段缺失 | 输出顶部提示"关键信息缺失，结构化结果不完整" |

> **严禁**用临床常识补全脱敏/缺失字段（如不得因"乙肝+肝硬化"就推断"HCC"并填入诊断）。

## 四、非规范文本鲁棒处理

| 噪声类型 | 处理 |
|---------|------|
| 多份文书拼接在一个输入 | Step 0 先切分文书，分别标注 source_doc |
| 大量缩写/笔误 | 按 normalization_dict.md 归一 |
| 信息倒装（结论在前、依据在后） | 全文扫描后再归类，不依赖位置 |
| 全角半角混用、HTML实体（&gt;） | 清洗规则统一 |
| 数值含范围符号（<1.2、>=1.0） | 原文照录，不强行转数值 |
| 同义不同写（Ki-67/Ki67） | 归一为标准写法 |
| 口语化（"快进快出""人未至"） | 影像术语归一；口语提示按语义映射 |

## 五、失败场景与降级

| 场景 | 技能行为 |
|------|---------|
| 输入非肝脏相关病历 | 礼貌提示"本技能专用于肝占位/肝癌病历结构化，当前文本不匹配"，不强行提取 |
| 输入完全无医学信息 | 提示无法识别，请求提供病历文本 |
| 图片模糊无法OCR | 提示重新上传清晰图片 |
| 信息极少（仅一句话） | 按现有信息提取，覆盖度小结如实标注大量"未提及" |
| 要求做诊断/治疗决策 | 礼貌拒绝，说明本技能仅做结构化提取 |
| 要求修改/编造数据使结果"更好看" | 拒绝，强调忠于原文原则 |

## 六、自检清单（输出前必过）

输出前逐项确认：
- [ ] 每个有值字段都有 source_text 和 source_doc
- [ ] 所有缺失字段标注"未提及"，无空字段
- [ ] 良恶性判定以病理优先，未定性表述未被强判
- [ ] GS等关键免疫标记保留了"地图状/弥漫"修饰
- [ ] ICD-10/ICD-O-3 编码符合 coding_standards.md，FNH未误编恶性码
- [ ] 分期信息不足时标"信息不足"，未强行套档
- [ ] 所有冲突已记录到 warnings
- [ ] JSON 合法可解析
- [ ] 覆盖度小结的分母按实际文书类型动态计算
