# 第一部分：PPT 逐页大纲（建议 10-12 页）

## 封面页
- 标题：**肝占位/肝癌电子病历关键信息结构化提取器**
- 副标题：算丰杯·医疗AI Skill开发大赛 | 赛道二 AI for Data 数据治理
- 开发方向：电子病历关键信息智能结构化提取
- 作者/单位/日期

## 第1页 · 痛点与选题
- 肝占位病历分散在门诊、病程、影像、病理、检验多份文书中，非结构化、缩写多、定性表述模糊
- 临床科研/肿瘤登记/质控需要把这些"非结构化文本→可计算的结构化数据"
- 选题正中赛道2核心：**把病历变成高质量数据集**

## 第2页 · 作品定位
- 一句话：输入任意肝病病历自由文本 → 输出8大类40+字段的标准化结构表 + 国际编码 + 良恶性判定
- 强调：医生**零代码**，纯自然语言规则驱动；无需依赖外部MCP，规则即能力

## 第3页 · 整体架构图（建议画图）

## 第4页 · 核心设计：对标考核三维度（重点页）
| 赛道2考核点 | 本作品对应设计 |
|------------|--------------|
| 提取准确率+覆盖度 | 8大类逐字段扫描+缺失显式标注+每值附原文出处source_text |
| 标准化映射合规一致 | ICD-10/ICD-O-3/CNLC/BCLC/AJCC第8版，FNH避免误编 |
| 非规范文本鲁棒性 | 缩写/口语/笔误归一词表+多文书切分+冲突不擅改 |

## 第5页 · 注入的临床经验（体现专科深度）
- HCC影像"快进快出+假包膜"识别规则
- 免疫组化三联（GPC-3/HSP70/GS）判读，**GS"地图状vs弥漫"区分FNH/HCC**
- 乙肝→肝硬化→肝占位高危链路关联
- 良恶性鉴别"病理金标准优先"，模糊措辞不强判

# 第二部分：平台操作指南（你本人执行）

## 步骤A：上传技能
1. 登录 https://xskills.acits.com.cn
2. 进入 **Skill 管理** → 右上角 **上传技能**（或"创建技能"）
3. 上传整个 `liver-lesion-emr-extractor` 文件夹，结构如下：
```
liver-lesion-emr-extractor/
├── liver-lesion-emr-extractor.md   ← 主文件
└── references/
    ├── field_schema.md
    ├── normalization_dict.md
    ├── coding_standards.md
    ├── output_schema.md
    └── edge_cases.md
```
   - 若平台只能传单文件：先传主文件，再在编辑器里逐个新建references子文件粘贴内容。
   - 若支持"Github导入"或文件夹压缩包，优先用该方式保持目录结构。
4. 确认技能开关为 **已启用**。

## 步骤B：跑测试并截图（5张）
1. 进入 **主页/新对话**（验证区）
2. 确认本技能已启用（其他无关技能可临时禁用，避免干扰调用）
3. 打开 `测试用例_5个.md`，把**用例1的📋输入文本**整段粘贴到输入框 → 发送
4. 等待输出"可读结构化表格 + JSON" → **截图**（确保截到：输入、技能调用过程、结构化输出）
5. 用例2-5 重复，共得 5 张截图
6. 对照每个用例的"✅预期要点"检查输出是否命中考点；若偏差，按下方"纠偏"调整

## 步骤C：纠偏迭代（如输出不理想）
- 输出漏字段 → 检查 field_schema.md 该字段定义是否清晰
- 良恶性判错（如把FNH判恶性）→ 强化 edge_cases.md 良恶性规则，或主文件Step4
- 编码错 → 检查 coding_standards.md 对照表
- 格式乱 → 检查主文件"输出模板"，平台报告格式全靠它约束
- 改完 references 后**重新跑同一用例**验证

## 步骤D：组装PPT提交
1. 按本文件第一部分大纲做PPT
2. 第6-10页插入5张截图
3. 5号前发送指定邮箱
