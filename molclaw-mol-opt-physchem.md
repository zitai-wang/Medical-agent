# 分子理化性质优化

> **注意**：本 skill 涉及的所有工具均已在 agent 工具集中通过标准 MCP 协议预配置，直接调用即可，无需编写连接脚本。

## 前置准备

- 若输入为本地文件，请先使用 `DrugSDA_Tool_upload_file` 上传至服务器
- 若输入为 PDB 文件，建议先用 `fix_pdb` 进行预处理

---

## Step 1：计算源分子性质

调用 `pred_mol_admet` 计算源分子的多项理化性质并生成汇总报告，重点关注用户查询中指定的目标性质。

---

## Step 2：基于 LLM 推理生成优化分子

根据 Step 1 的性质分析报告和以下提示词，利用 LLM 推理能力从源分子生成优化分子，并提供详细的优化理由。

---

### 角色定义

你是一位拥有 15+ 年先导化合物优化和药物设计经验的资深药物化学专家，擅长优化小分子药物的理化性质，同时保持结构完整性和合成可行性。

### 任务概述

优化源分子以改善特定理化性质（LogP、QED 或溶解度），遵循药物发现最佳实践。你必须提供：
1. 结构化的优化推理过程
2. 最终优化分子的 SMILES 格式
3. 每项修改的明确理由

### 背景知识

**Lipinski 五规则 (RO5)**

| 性质 | 最佳范围 |
|------|---------|
| 分子量 (MW) | < 500 Da |
| LogP（亲脂性） | -0.4 ~ 5.0 |
| 氢键供体 (HBD) | ≤ 5 |
| 氢键受体 (HBA) | ≤ 10 |
| 可旋转键 | ≤ 10 |

**QED（药物相似性定量评估）**
- 评分范围 0~1（越高越好，>0.67 为理想值）
- 关键因素：MW、LogP、HBA、HBD、TPSA、可旋转键、芳香环数
- 优化策略：平衡所有因素，而非最大化单一指标

**溶解度优化策略**

| 策略 | 化学修饰 |
|------|---------|
| 增加极性基团 | -OH, -NH₂, -COOH, -SO₃H |
| 降低亲脂性 | 移除芳香环、烷基链 |
| 引入可电离基团 | 胺基、羧酸 |
| 降低分子量 | 移除非必需取代基 |
| 破坏晶体堆积 | 增加支链、降低对称性 |

**LogP 优化策略**

| 目标 | 策略 | 示例修饰 |
|------|------|---------|
| ↓ LogP | 添加极性基团，移除疏水基团 | -CH₃ 替换为 -OH |
| ↑ LogP | 添加芳香环、烷基链、卤素 | 添加 -Ph, -Cl, -CF₃ |

### 优化工作流

**阶段 A：分析源分子**
- 审查 Step 1 中 `pred_mol_admet` 计算的性质值
- 定位结构中有问题的特征
- 识别不会破坏核心药效团的修饰位点
- **立体化学检查**：若源 SMILES 包含立体化学标记（`@`、`@@`、`/`、`\`），列出所有手性中心和顺反键，这些标记必须在输出中保持不变（除非修饰直接涉及该立体中心）

**阶段 B：设计修饰策略**
- 选择 1-2 个具体的结构变更
- 用药物化学原理证明每项修改的合理性
- 考虑合成可行性

**阶段 C：验证优化分子**
- 确保 SMILES 化学上有效
- 验证修饰与目标性质改善一致
- 检查是否引入了严重的药物相似性违规
- **立体化学验证**：确认所有原始 `@`、`@@`、`/`、`\` 标记均存在且未改变

### 优化示例

**示例 1：溶解度优化 — 生物电子等排环替换（苯 → 吡啶）**

源分子: `c1ccc(-c2ccc(NC(=O)C)cc2)cc1`

```json
{
  "Analysis": "联苯核心导致溶解度差，LogP ~2.9。高平面性和对称性促进晶体堆积。",
  "OptimizationStrategy": "将一个苯环替换为吡啶：'c1ccccc1' → 'c1ccncc1'。增加 HBA，降低 LogP ~0.4，破坏对称性。",
  "Final Target Molecule": "c1ccnc(-c2ccc(NC(=O)C)cc2)c1",
  "ExpectedImprovement": "LogP ~2.9 → ~2.5。通过增加 HBA 和破坏对称性改善溶解度。形状保持。",
  "Confidence": "High"
}
```

**示例 2：失败案例与修正**

源分子: `c1ccc2c(c1)cc(NC(=O)C)c1ccccc12`（氨基芴酰胺）
目标：降低 LogP

❌ 第一次尝试（失败）:
```json
{
  "Analysis": "芴三环系统导致高 LogP (~3.5)。",
  "OptimizationStrategy": "在芴环上加 -OH 并插入环氮。",
  "Final Target Molecule": "c1ccc2c(c1)cc(NC(=O)C)c1cc(O)ccc12N",
  "ExpectedImprovement": "预期 LogP 降低。",
  "Confidence": "Medium"
}
```
**验证**：❌ 无效 — 在环闭合外追加 `N` 破坏了桥环编号。同时进行两项修改增加了错误风险。

✅ 第二次尝试（修正）:
```json
{
  "Analysis": "LogP 高 (~3.5) 由芴三环系统导致。保持骨架完整。",
  "OptimizationStrategy": "仅做一项修改：在第一个苯环上添加 -OH。在环遍历路径中使用 '(O)' 插入。",
  "Final Target Molecule": "c1cc(O)c2c(c1)cc(NC(=O)C)c1ccccc12",
  "ExpectedImprovement": "LogP ~3.5 → ~2.9。增加 1 HBD + 1 HBA。骨架保持。",
  "Confidence": "High"
}
```
**教训**：(1) 稠环系统每次只做一项修改。(2) 绝不在环闭合外追加原子 — 使用 `()` 在环路径内。(3) 优先选择更简单的修改。

### 关键约束

⚠️ **禁止**：
- 破坏对活性至关重要的核心药效团结构
- 创建化学上不稳定或不可能的结构
- 引入有毒官能团（硝基芳烃、活性环氧化物等）
- 生成语法错误的 SMILES
- 同时违反多项 RO5 标准
- 丢失或修改源分子 SMILES 中的立体化学标记

✅ **必须**：
- 尽可能保持骨架完整性
- 移除官能团时使用生物电子等排体替换
- 考虑合成可行性
- 为每项修改提供清晰的理由
- 确保输出 SMILES 有效且可解析
- 保留源分子中的所有立体化学信息

### 输出格式

回答必须是合法的 JSON 格式：

```json
{
  "Analysis": "对源分子性质和问题的简要分析",
  "OptimizationStrategy": "带理由的分步修饰方案",
  "Final Target Molecule": "优化分子的有效 SMILES 字符串",
  "ExpectedImprovement": "预期性质改善说明",
  "Confidence": "High / Medium / Low"
}
```

## Step 3：验证分子有效性

调用 `is_valid_smiles` 验证 Step 2 生成分子的结构完整性。若分子无效，分析根本原因，反思错误，重新执行 Step 2（**不要重复导致无效 SMILES 的相同修改策略**）。若有效，直接进入 Step 4。

## Step 4：计算优化分子性质并比较

调用 `pred_mol_admet` 计算新生成分子的多项理化性质，并与源分子进行严格比较。目标是**最大化改善**目标性质，追求显著提升而非边际调整。

**注意**：若用户指定了特定的优化目标，**以用户指定的目标为准**。

**严格改善阈值（默认目标）**：

| 性质 | 阈值要求 |
|------|---------|
| LogP | 向目标方向变化 ≥ **2.5** |
| QED | 提升 ≥ **0.5** 或达到至少 **0.9** |
| 溶解度 (LogS) | 提升 ≥ **4.0** |

若目标性质未达到上述阈值，进行深度根因分析并批判性地反思优化策略。必须**以完全不同的策略**重新执行 Step 2。**任何情况下都不得**重复之前未能达成显著改善的策略。
